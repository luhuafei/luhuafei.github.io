IOS面试试题

发表于 2017-10-01 | 分类于 试题

从网上看到很多面试试题, 总结一些自己觉的不错的.

为什么说OC是一门动态语言

oc类的类型和数据变量的类型都是在运行时确定的,而不是在编译时确定的,例如多态性: 多态特性,
我们可以使用父类对象来指定子类对象, 并且可以用来调用子类的方法.运行时特性, 我们可以动态
的添加方法或者替换方法.

为什么代理要用weak? 代理的delegate和dataSource有什么区别? block和代理的区别?

代理是用weak修饰的:
   1使用weak是为了避免循坏引用,
   2.当使用weak修饰的属性, 当对象释放的时候, 系统会对属性赋值nil, Object-C有个特性
     就是对nil对象发送消息时, 不会crash.
delgate表示代理, 代理可以让A对象通知B对象, A发生变化, 前提是B遵循了A的代理, 并且实现了A的
方法.
dataSource表示数据源, 如果A对象声明了数据源, 当我们创建A对象的时候,我们就该实现数据源, 来告诉A
他所需要的一些数据.例如:tableview数据源方法, 需要告诉它, 我要实现几组cell, 每组cell返回多少行,
实现的cell什么样子.
delgate和datasource都可以使用require和optional来修饰
代理和block的区别
  相同点:
        1.代理和block大多都可以用来做倒序传值的,
        2.注意避免循坏引用.
        3.调用的时候都要判断是否已经实现.
  不同点:
        1.代理必须先声明方法, block则是保存的一段代码, 其实就是一个函数,
        2.修饰不一样, 代理用weak修饰, block用copy修饰block则是保存的一段代码

为什么IBOutlet修饰的控件使用weak来修饰的?

在xib或者sb中的拓控件时, 其实控件就加载到了父控件的subviews数组里面了, 已经进行了强引用,
即weak, 也不会造成对象的释放.

Runtime

objc在向一个对象发送消息时,发生了什么?

1.根据对象的isa指针找到类对象id, 在查询类对象里面的methodLists方法函数列表, 
 2.如果没有找到, 再沿着superClass寻找父类, 再在父类的methodLists方法列表里查询, 
 3.最终找到SEL, 根据id和 SEL确认IMP(指针函数), 再发送消息.

什么时候会报unrecognized selector错误? ios有哪些机制避免走到这一步?

1.当发送消息的时候, 我们会根据类里面的methodLists列表去查询我们要动用的SEL, 
当查询不到时候, 我们会直沿着父类查询, 

2.当最终查询到的时候我们会报unrecognized selector错误, 当系统查询不到方法的时候,会调用 +  (BOOL)resolveInstanceMethod:(SEL)sel
 动态解释的方法来给我一次机会来添加,调用不到的方法. 

3或者我们可以再次使用- (id)forwardingTargetForSelector(SEL)sSelector                      
 重定向的方法来告诉系统, 该调用什么方法,一来保证 不会崩溃.

能否向编译后得到的类中增加实例变量? 能否向运行时创建的类中添加实例变量? 为什么?

1.不能向编译后得到的类增加实例变量
2.能向运行时创建的类中添加实例变量
解释:
  1.编译后的类已经注册在runtime中, 类结构体中的objc_ivar_list 实例变量的链表和 instance_size实例变量
      的内存大小已经确定, runtime会调用class_setvarlayout 或 class_setWeaklvarLayout来处理strong 和 
      waek引用, 所以不能向存在的类中添加实例变量.
   2,运行时创建的类是可以添加实例变量的, 调用class_addIvar函数, 但是在调用objc_allocateClassPair之后, 
       objc_registerClassPair之前, 原因同上.

runtime如何实现weak变量的自动置nil?

1.runtime对注册的类, 会进行布局, 对于weak对象会放入一个hash表中. 用weak指向的对象内存地址作为 
   key, 当此对象的引用计数为0的时候回dealloc.

 2.假如weak指向的对象内存地址是A, 那么就会以A为键, 在这个weak表中搜索, 找到所有以A为键的weak
    对象, 从而置为nil.

给类添加一个属性后, 在类结构体里那些元素会发生变化

1.instance_size: 实力内存的大小
2.objc_ivar_list *ivars : 属性列表
